这里列出所有文档化的文件，附带简要说明：
libtaomee/atomic.h从内核代码里抽取出来的整型原子操作函数/宏。
libtaomee/hash_algo.h哈希算法
libtaomee/list.h从内核代码里抽取出来的侵入式链表操作函数/宏。
Simple doubly linked list implementation.
libtaomee/log.h用于记录日志，一共分9种日志等级。日志文件可以自动轮转。注意：如果使用自动轮转，
必须保证每天写的日志文件个数不能超过log_init时设定的最大文件个数，否则日志会写乱掉。 必须先调用log_init来初始化日志功能。注意，每条日志不能超过8000字节。
如果编译程序时定义宏LOG_USE_SYSLOG，则会利用syslog来记录日志，使用的facility是LOG_USER。
libtaomee/timer.h定时器函数，有秒级和毫秒级两种精度的接口。用于设定某一时刻调用某个函数。需要glib支持。
libtaomee/utils.h一些常用的函数。
libtaomee/bitmanip/bitmanip.h位操作函数
libtaomee/conf_parser/config.h用于解析配置文件。配置文件的格式为：
============================================
#key value | #号打头的行被视为注释
log_dir ./log
log_size 32000000
============================================
libtaomee/crypt/qdes.hDES ECB加密
libtaomee/dataformatter/bin_str.h用于把数值转换成字符串形式，或者把字符串形式的数字转换成数值。
libtaomee/dirtyword/dirtyword.h该模块提供了脏词检测的接口，包括读入脏词库，创建搜索结构，以及进行脏词检测等操作。 该模块需要一个配置文件，用来指定脏词库的位置及一些内存映射文件的信息，配置文件的说明详见doc/readme.txt
libtaomee/inet/ipaddr_cat.hIp地址分类
libtaomee/inet/mcast.h多播（组播）相关函数。为了更好地理解和使用这些函数，建议认真阅读一下APUE2第21章 - Multicasting。
libtaomee/inet/tcp.hEssential TCP socket interface
libtaomee/interprocess/filelock.h用于操作文件锁。注意事项：
1. 关闭文件会导致该文件的所有文件锁都被解锁。假设fd1和fd2 对应的都是同一个文件，那么无论关闭了fd1或者fd2，都会导致 该文件的所有文件锁被解锁。
2. 子进程不能继承父进程获得的文件锁。
3. 通过exec来执行新的程序的话，这个新的程序可以继承原来的 程序的文件锁。但是，如果原来的文件对应的fd被设置了close-on-exec 位的话，则不能继承原来程序的文件锁。
Advanced Programming in the UNIX Environment - 14.3. Record Locking 中有更详细的说明，建议认真阅读一下。
libtaomee/interprocess/shmq.h共享内存队列。目前的shm_cq只适用于一个进程只读，另外一个进程只写的应用。
具体例子请参见/samples/shmq/
libtaomee/project/constants.h定义淘米项目公共常量。
libtaomee/project/types.h定义淘米项目公共类型。
libtaomee/project/utilities.h淘米项目公共小函数。
libtaomee/project/stat_agent/msglog.h用于写入统计信息
libtaomee/random/random.h用于产生随机数。
libtaomee/time/time.h和时间相关的函数。部分函数需要glib支持。
